* 플레이어들을 경주마로 표현하고 랜덤하게 진행시켜 승자를 뽑는 시뮬레이션 게임 웹앱이다. 
* 프로젝트는 심플하게 구성하여 index.html, join.html, style.css, app.js 으로 이루어진다.
* 게임 세션은 Firebase를 연동해서 플레이어들을 응원하면 어드밴티지를 주는 것이 특징이다.


1. UI
UI는 상단 영역과 그 아래 대부분을 경기장 영역으로 한다.
- 상단: 플레이어 이름 입력 필드, 게임 시작 버튼
- 경기장: 화면의 90% 너비. 플레이어 숫자만큼 트랙이 위에서부터 아래로 순서대로 나열됨
- 트랙: 왼쪽부터 플레이어 이름, 출발선, 경기구간이 길게 존재, 결승점, 도착순서(등수)
- 경기구간: 경마 이모지 🏇를 좌우반전하여 표시, 진행에 따라 왼쪽에서 오른쪽으로 애니메이션으로 이동
- 카운트다운: 경기장 중앙에 표시되며, modal 스타일로 크고 굵은 폰트로 표시
- 결과 팝업: 경기가 종료되면 도착순서대로 1등부터 플레이어 이름을 순위와 함께 팝업으로 표시.

2. 게임의 흐름
- 플레이어 이름은 ,로 구분하여 입력받는다. 2~10명까지 입력가능하며 모든 이름은 unique하다.
- 게임 시작 버튼을 누르면 경기장을 초기화하여 표시한다.
- 경기장을 초기화하면 카운트다운이 5초부터 표시되어 시작한다.
- 카운트다운이 완료되면 각 경주마들이 결승점을 향해 진행한다.
- 한 틱은 1초이며 한 틱장 진행량은 각각 랜덤하다.
- 경주마가 결승점에 도달하면 도착순서(등수)를 표시한다.
- 모든 경주마가 결승점에 도달하면 결과 팝업을 표시한다.

3. 게임세션과 Firebase를 연동
- 이 프로젝트는 Firebase Realtime Datebase를 연동한다. 연동포인트는 https://kooksun-hr-default-rtdb.firebaseio.com/ 이다.
- 게임 시작 버튼을 누르면 db에 플레이어 이름과 응원수를 key-value로 하는 자료구조를 생성한다.
- 웹페이지가 언로딩되거나 게임이 종료되면 db를 지운다.
- index.html이 처음 로딩되고 나면 db를 열어본다.
만약 db가 없으면, 게임을 호스팅 하는 모드이다.
만약 db가 있으면 이미 게임이 시작된 상태로 응원을 하는 모드이다.
- 응원 모드에서는 단순히 플레이어의 이름이 버튼으로 나열된다.
- 플레이어의 버튼을 누르면 db에 있는 플레이어에 해당하는 응원수가 +1 된다. 
- 플레이어별 응원수는 게임의 진행에서 틱당 랜덤한 진행량에 덧붙여서 더 빠른 진행을 할 수 있게 한다.

4. 기타
- 게임의 진행(시작/진행중/종료)내용을 콘솔 로그에 출력한다
특히 매 틱당 각 플레이어의 진행량을 출력한다
- Firebase의 연동과 관련된 내용을 콘솔 로그에 출력한다
특히 매 틱당 db에서 연동하는 응원수를 출력한다
- 경주마의 진행에 대해서는 구현 이후에 테스트하며 정도를 조정할 것이다
